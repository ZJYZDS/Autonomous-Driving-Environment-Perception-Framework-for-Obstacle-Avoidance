# 不同尺度物体共享 2D 网络的影响分析

**日期**: 2026-07-10

## 问题

不同尺寸的 YOLO 检测框统一 resize 到 128×128 后输入同一个 2D extractor (41K)。
远距离小物体（如 60×30 px 的车）被放大，近距离大物体（如 300×600 px 的人）被压缩。
同一个轻量网络能否处理这种尺度差异？

## 影响评估

### 反面: 有效分辨率不同

| 物体 | 原始尺寸 | resize 操作 | 等效质量 |
|------|---------|------------|---------|
| 近处行人 | 300×600 px | 压缩 4.7x | 细节丰富 |
| 远处卡车 | 60×30 px | 放大 2.1x | 模糊、锯齿 |
| 中距轿车 | 100×60 px | 轻微放大 | 正常 |

同一网络参数要在三种输入质量下工作。

### 正面: 任务不是精测而是判别

2D 分支的任务是辅助 3D 主干，而非独立预测。它只需要回答:

- 这是车头还是车尾? (yaw 歧义)
- 有没有被遮挡? (截断推断)
- 是轿车还是卡车? (尺寸先验)

这些判别信号是**低频特征**:

- 车灯的相对位置不因 resize 而改变
- 物体长宽比在 128×128 下仍然可辨
- 颜色分布(红色尾灯 vs 白色头灯)对分辨率不敏感

### 真正失效的场景

1. **极近处截断**: 只能看到车门局部, 不包含朝向信息 → 输入本身无判别性
2. **极远处退化**: 10 px 的 blob, resize 后纯噪声 → YOLO 检测本身不可靠

这两种情况的瓶颈是 YOLO 输入质量, 不是 2D 网络结构。

## 如果后续需要解决

参照 3D 分支的 extent 归一化思路, 向 2D 分支注入尺度先验:

```python
# 在 conv stem 输出后, 每个 token 拼接 bbox 尺度信息
bbox_scale = torch.stack([log(bbox_w), log(bbox_h)], dim=-1)  # (B, 2)
scale_feat = scale_proj(bbox_scale).unsqueeze(1).expand(-1, 64, -1)  # (B, 64, D)
tokens = tokens + scale_feat  # 或 concat
```

额外参数: Linear(2, 48) ≈ 144 个参数, 可忽略。

## 当前建议

先不引入尺度调制。在全量训练后按物体距离/尺寸分层评估:

- 如果不同距离区间误差分布均匀 → 尺度不敏感, 保持原样
- 如果远距离系统偏大 → 考虑注入尺度特征

70% 凭实验, 30% 凭推演。
