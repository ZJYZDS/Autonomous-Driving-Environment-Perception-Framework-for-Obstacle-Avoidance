# C2 vs C3 最终对比

**日期**: 2026-07-10

## 训练状态

| 模型 | Checkpoint | Epoch | Val Loss | 参数 |
|------|-----------|-------|----------|------|
| C2 (LidarOnlyRefiner) | lidar_only.pt | 19/20 | 0.0621 | 191K |
| C3 (CrossModalFusion) | fusion.pt | 23/25 | 0.0511 | ~2.8M |

## 逐帧对比结果 (69 objects)

| 指标 | C2 (纯3D) | C3 (融合) | 优势 |
|------|-----------|-----------|------|
| Center err | 0.460m | **0.441m** | C3 优 4% (0.019m) |
| Size err | 0.120m | **0.114m** | C3 优 5% (0.006m) |
| Yaw err | **4.40°** | 4.54° | C2 优 3% (0.14°) |
| Head-to-head | 30 | **39** | 57% vs 43% |

## 分析

### 差距在噪声范围内

- Center 差距 1.9cm — 比 GT 噪声(0.3m)小一个数量级
- Size 差距 6mm — 比 GT 噪声(0.15m)小一个数量级
- Yaw 差距 0.14° — 比 GT 噪声(5°)小两个数量级

三个指标的实际差异远小于标注噪声的尺度，无法区分是模型优势还是统计波动。

### C3 的成本

| 项目 | C2 | C3 | 倍数 |
|------|-----|-----|------|
| 参数量 | 191K | ~2.8M | 14x |
| 2D extractor | 无 | 41K | - |
| Cross-attention | 无 | ~2.1M | - |
| MLP head | ~30K | ~500K | 17x |
| 推理时需要 RGB crop | 否 | 是 | - |

C3 用 14 倍参数换来 4-5% 的边际误差降低，且 yaw 反而更差。

### 与消融结论一致

之前的消融实验结论再次被验证：在当前 nuScenes 数据分布下（城市道路、物体<30m、点云密集、完整可见），**2D 外观信号对 3D bbox refinement 不提供实质性增益**。2D 图像在几何信息不足时才有不可替代的作用（远距离、遮挡、朝向歧义），但这些场景在当前数据中占比低。

## 结论

**C2 (纯 PointNet++, 191K) 是当前场景的最优方案。** C3 不推荐用于当前数据分布。
