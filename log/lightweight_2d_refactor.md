# 2D 分支轻量化重构

**日期**: 2026-07-10

## 背景

消融实验证实 C2 (纯3D, 191K) ≈ C1 (纯2D, 2.89M)，2D 图像在几何饱和场景下不提供额外增益。但 2D 分支仍应保留作为下限保障（远距离/遮挡/朝向歧义）。

然而原架构存在严重参数预算倒挂：辅助 2D 分支 (ResNet18, 2.73M) 是主干 3D 分支 (PointNet++, 191K) 的 **14 倍**。辅助方比主唱声音大得多，在架构上不合理。

## 重构思路

### 核心原则

1. **辅助信号参数量应远小于主干** — 否则它不是辅助，而是噪声源
2. **检测框内物体外观多样性 << ImageNet** — 不需要预训练 backbone
3. **无残差连接的纯卷积，堆深反而退化** — 浅层即可

### 三层递进

| 轮次 | 结构 | 2D 参数 | 2D/3D 比 | 分析 |
|------|------|---------|----------|------|
| 原始 | ResNet18 (pretrained) | 2,733K | 14.0x | 严重倒挂 |
| 第一版 | 4 conv (s2×4) + self-attn(d=128) | 110K | 0.74x | 仍偏大 |
| 第二版 | 3 conv (s4,2,2) + self-attn(d=64) | 75K | 0.50x | 可接受 |
| 最终版 | 2 conv (s8,2) + self-attn(d=48) | 41K | 0.27x | 理想 |

### 最终架构 (Lightweight2DExtractor)

```
输入: (B, 3, 128, 128)   YOLO 检测框内的 RGB crop

Conv1: 3→16, k=7, s=8, p=3  → (B, 16, 16, 16)   # 大核大跨度, 一次看清物体全局
Conv2: 16→48, k=3, s=2, p=1 → (B, 48, 8, 8)     # 局部细调

Reshape: (B, 48, 8, 8) → (B, 64, 48)              # 64 个 2D token

Self-Attn (d=48, n_heads=4, ffn_expand=2)         # 建模 token 间外观关系
  → MHA + residual + FFN(GELU)
  → (B, 64, 48)

Proj: Linear(48, 256) → (B, 64, 256)              # 投影到 d_model 供 cross-attn
```

### 设计理由

- **2 层 conv 够用**: 检测框内只有一辆车/行人，不是"任意照片"，2 层足以提取车头/车尾/车侧的外观差异
- **stride=8 首层 + k=7 大核**: 128×128 一次下采样到 16×16，大核看 ~45px 范围，覆盖物体关键部件（车灯、车窗）
- **d=48 极窄内通道**: 仅为 d_model(256) 的 19%。外观判别不需要高维，车/人/骑车人的 visual feature 低维即可分离
- **self-attn 在窄空间**: MHA(d=48) 仅 9K 参数，FFN 9K，总计 18K。在投影前做 attention 比 d=256 省 15 倍

## 最终参数分布 (CrossAttnRefiner C3)

| 模块 | 参数量 | 占比 | 角色 |
|------|--------|------|------|
| 2D extractor | 41K | 1.5% | 辅助: 外观特征 |
| 3D encoder (PointNet++) | 149K | 5.4% | 主干: 几何定位 |
| Cross-Attn ×2 + Proj | ~2.1M | 76% | 融合: 跨模态信息交换 |
| MLP Head | ~500K | 18% | 回归: bbox 残差 |

辅助:主干 = 0.27x，彻底解决参数倒挂问题。
